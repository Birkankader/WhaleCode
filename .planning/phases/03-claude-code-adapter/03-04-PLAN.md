**useClaudeTask.ts changes:**

1. **Enhanced silent failure detection:** Instead of only checking for `"is_error":true` string match, track the last result event. In the `channel.onmessage` handler for stdout events:
   - Try to JSON.parse each stdout line
   - If it parses and has `type === "result"`, store it as `lastResultJson` (the raw string, not parsed)
   - On `exit` event, if `lastResultJson` exists, call `commands.validateClaudeResult(lastResultJson)`. If it returns an error, set `silentFailure` to true and override status to `failed` regardless of exit code.
   - If exit code is 0 but NO result event was ever received, also flag as silent failure.

2. **Rate limit retry loop:** When a rate limit is detected in stderr:
   - Track `retryAttempt` (start at 0, max 3)
   - If `retryAttempt < 3`, cancel the current process via `commands.cancelProcess(taskId)`, wait for `5000 * 2^attempt` ms (capped at 60000), then call `spawnTask` again recursively (or use a loop)
   - Update the rate limit banner text to show retry progress: "Rate limited. Retrying in {delay}s (attempt {n}/3)..."
   - If all retries exhausted, set banner to "Rate limit retry exhausted after 3 attempts" and leave task as failed

   Implementation: Add `retryAttempt` ref. When rate limit detected in stderr handler, set a flag. In the exit handler, check if rate limit was detected during this run. If so, and retryAttempt < 3, increment attempt, sleep, and re-invoke the spawn. Use a ref for the retry state to avoid stale closures.

   Simpler approach: Extract the core spawn logic into an inner function. The outer `spawnTask` calls it in a loop:
   ```typescript
   const spawnTask = useCallback(async (prompt, projectDir) => {
     const policy = { maxRetries: 3, baseDelay: 5000, maxDelay: 60000 };
     for (let attempt = 0; attempt <= policy.maxRetries; attempt++) {
       if (attempt > 0) {
         const delay = Math.min(policy.baseDelay * 2 ** (attempt - 1), policy.maxDelay);
         setRateLimitWarning(`Retrying in ${delay/1000}s (attempt ${attempt}/${policy.maxRetries})...`);
         await new Promise(r => setTimeout(r, delay));
       }
       const { taskId, hitRateLimit } = await spawnOnce(prompt, projectDir);
       if (!hitRateLimit) return taskId;
       // Rate limit hit — cancel and retry
       if (taskId) await commands.cancelProcess(taskId);
     }
     setRateLimitWarning('Rate limit retry exhausted after 3 attempts');
     return null;
   }, []);
   ```

   Change `rateLimitWarning` state from `boolean` to `string | false` to carry retry status messages. Update ProcessPanel if it reads `rateLimitWarning` as boolean (it should still render truthy values).

   The `spawnOnce` inner function contains the current spawn logic but resolves a promise when the process exits, returning `{ taskId, hitRateLimit, silentFailure }`.

3. **Call validateClaudeResult on exit:** In the exit handler within spawnOnce, if `lastResultJson` ref is set, call `commands.validateClaudeResult(lastResultJson)`. On error result, mark as silent failure.
