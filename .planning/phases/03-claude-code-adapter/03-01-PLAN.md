    2. Create `src-tauri/src/credentials/mod.rs` with `pub mod keychain;`.

    3. Create `src-tauri/src/credentials/keychain.rs` implementing:
       - `const SERVICE_NAME: &str = "com.whalecode.app";`
       - `const CLAUDE_API_KEY_USER: &str = "anthropic-api-key";`
       - `pub fn get_api_key() -> Result<String, String>` — uses `Entry::new(SERVICE_NAME, CLAUDE_API_KEY_USER)?.get_password()`
       - `pub fn set_api_key(key: &str) -> Result<(), String>` — uses `entry.set_password(key)`
       - `pub fn delete_api_key() -> Result<(), String>` — uses `entry.delete_credential()`
       - `pub fn has_api_key() -> bool` — returns `get_api_key().is_ok()`
       - All functions map keyring errors to String with descriptive messages.
       - IMPORTANT: These are synchronous functions. Callers in async context MUST use `tokio::task::spawn_blocking` (see research Pitfall 4).

    4. Add `mod credentials;` to `src-tauri/src/lib.rs`.

    5. Write unit tests in `credentials/keychain.rs` `#[cfg(test)] mod tests`. Note: tests interact with real macOS Keychain so use a unique test service name `com.whalecode.test` to avoid polluting real credentials. Add a test helper that uses a different service name. Alternatively, mark integration-style keychain tests with `#[ignore]` and test the logic with a mock — but prefer real keychain tests since this is macOS-only.
  </action>
