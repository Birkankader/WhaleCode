    - Test: optimize("fix bug", "gemini", context) produces output starting with context, NOT containing "## Task Plan"
    - Test: optimize("fix bug", "claude", context) != optimize("fix bug", "gemini", context) for same input
    - Test: optimize_all("fix bug", context) returns Vec with exactly 2 entries (claude + gemini)
    - Test: context with recent_events includes event summaries and file paths in output
    - Test: empty context (no events) produces prompt with just the user task, no empty context section
    - Test: optimized prompt total length stays under 8000 chars even with many events (truncation)
  </behavior>
  <action>
    Create `src-tauri/src/prompt/` module with three files:

    **models.rs**: Define types needed by the prompt engine:
    - `PromptContext` struct (NOT specta-exported, internal only): `recent_events: Vec<ContextEventSummary>`, `project_dir: String`
    - `ContextEventSummary` struct: `tool_name: String`, `event_type: String`, `summary: String`, `files: Vec<String>`, `created_at: String`
    - `OptimizedPrompt` struct (Serialize + specta::Type for IPC): `tool_name: String`, `original_prompt: String`, `optimized_prompt: String`

    **templates.rs**: Two template functions + two context formatters:
    - `claude_template(prompt, context) -> String`: Produces structured prompt with:
      1. "## Task Plan\n\nBefore making changes, analyze the codebase and plan your approach." planning preamble
      2. "## Project Context" section with recent events (if any), formatted with timestamps, tool names, summaries, file lists, ending with "Consider these recent changes when planning your approach."
      3. "## Task\n\n{prompt}" section
      Sections joined with "\n\n---\n\n"
    - `gemini_template(prompt, context) -> String`: Produces flat prompt with:
      1. "Context: Recent project changes:" section with events listed as "- {summary} by {tool_name} at {created_at} [{files}]" (if any)
      2. "Task: {prompt}" direct instruction
      Sections joined with "\n\n"
    - `default_template(prompt, context) -> String`: Falls back to claude_template (safe default)
    - Helper: `format_context_for_claude(context) -> String` and `format_context_for_gemini(context) -> String`
    - Apply max 8000 char limit on total optimized prompt (truncate context events, not user prompt)

    **mod.rs**: PromptEngine struct with two public methods:
    - `optimize(raw_prompt, tool_name, context) -> OptimizedPrompt`: Match tool_name to template function
    - `optimize_all(raw_prompt, context) -> Vec<OptimizedPrompt>`: Call optimize for "claude" and "gemini"
    - Add `pub fn build_prompt_context(context_store, project_dir) -> Result<PromptContext, String>` helper that:
      1. Calls `context_store.with_conn(|conn| get_recent_events(conn, project_dir, 5))`
      2. Maps `(ContextEvent, Vec<String>)` tuples to `ContextEventSummary` structs
      3. Returns `PromptContext { recent_events, project_dir }`
    - Include `#[cfg(test)] mod tests` with all behavior tests listed above

    Add `mod prompt;` to `src-tauri/src/lib.rs` (just the module declaration, between existing mods).
