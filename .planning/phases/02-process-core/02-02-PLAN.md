**src/hooks/useProcess.ts** — React hook managing process lifecycle:
- `useProcess()` returns `{ processes, spawnProcess, cancelProcess, pauseProcess, resumeProcess }`
- `processes` is a Map<string, { status: 'running' | 'paused' | 'completed' | 'failed', exitCode?: number }>
- `spawnProcess(cmd, args, cwd)`: creates Channel, calls `commands.spawnProcess`, stores task_id, wires Channel.onmessage to update state and forward events
- `cancelProcess(taskId)`: calls `commands.cancelProcess`, updates local state
- `pauseProcess(taskId)`: calls `commands.pauseProcess`, updates local state
- `resumeProcess(taskId)`: calls `commands.resumeProcess`, updates local state
- Use zustand or React useState — Claude's discretion (zustand preferred for cross-component access)

**src/components/terminal/OutputConsole.tsx** — extend existing component:
- Accept optional `processId: string` prop
- Accept optional `channelRef` callback to expose Channel to parent
- Add timestamp prefix to each output line: `[HH:MM:SS]` format before the line content
- Keep all existing functionality (xterm.js, FitAddon, ResizeObserver)
- When processId provided, don't auto-start stream (parent manages Channel lifecycle)

**src/components/terminal/ProcessPanel.tsx** — tabbed container:
- Renders a tab bar showing each process (from useProcess hook)
- Each tab shows process status icon (running=green dot, paused=yellow, completed=gray, failed=red)
- Active tab shows that process's OutputConsole
- Control buttons per process: Cancel (red), Pause/Resume (toggle), visible in tab or toolbar
- "Spawn Test Process" button for development: spawns `echo "hello" && sleep 5 && echo "done"` (remove before Phase 3)
- Uses Tailwind for styling (dark theme matching existing app)

**src/App.tsx** — integrate ProcessPanel:
- Add ProcessPanel to the main layout alongside or replacing the existing single OutputConsole
- Keep existing routing structure

IMPORTANT: Use `.catch()` on all IPC calls for non-Tauri environments (per Phase 1 pattern). Timestamp format: `[HH:MM:SS]` using `new Date().toLocaleTimeString('en-US', { hour12: false })`.
