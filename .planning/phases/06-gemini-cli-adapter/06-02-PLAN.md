  <behavior>
    - parseGeminiEvent parses init event with session_id and model
    - parseGeminiEvent parses message event with role and content (string, not array)
    - parseGeminiEvent parses tool_use event with tool_name and parameters
    - parseGeminiEvent parses tool_result event with tool_id, status, output
    - parseGeminiEvent parses result event with status, response, stats (total_tokens, duration_ms)
    - parseGeminiEvent parses error event with message
    - parseGeminiEvent returns null for non-JSON and empty strings
    - formatGeminiEvent formats init as "[Session started] model={model}"
    - formatGeminiEvent formats message as the content string directly
    - formatGeminiEvent formats tool_use as "[Tool: {tool_name}] {JSON.stringify(parameters)}"
    - formatGeminiEvent formats tool_result as "[Result] {output}"
    - formatGeminiEvent formats error as "[Error] {message}"
    - formatGeminiEvent formats result as "[Done] tokens={total_tokens}, duration={duration_ms}ms"
    - formatGeminiEvent returns raw line for unparseable input
  </behavior>
  <action>
    Create `src/lib/gemini.ts` mirroring claude.ts but with Gemini-specific types:

    1. **GeminiStreamEvent interface**:
       - type: 'init' | 'message' | 'tool_use' | 'tool_result' | 'result' | 'error'
       - session_id?, model?, timestamp? (init)
       - role?, content?: string (message -- NOTE: string not array, unlike Claude)
       - tool_name?, tool_id?, parameters?: unknown (tool_use -- field names differ from Claude: tool_name not name, parameters not input)
       - output?, status? (tool_result)
       - message? (error)
       - response?, stats?: { total_tokens?, input_tokens?, output_tokens?, duration_ms?, tool_calls? } (result)

    2. **parseGeminiEvent**: Same pattern as parseClaudeEvent -- JSON.parse, check for 'type' field, return null on failure.

    3. **formatGeminiEvent**: Format each event type for terminal display:
       - init: `[Session started] model={event.model ?? 'unknown'}`
       - message: `event.content ?? ''`
       - tool_use: `[Tool: ${event.tool_name}] ${JSON.stringify(event.parameters)}`
       - tool_result: `[Result] ${event.output ?? ''}`
       - error: `[Error] ${event.message ?? 'Unknown error'}`
       - result: `[Done] tokens=${event.stats?.total_tokens ?? '?'}, duration=${event.stats?.duration_ms ?? '?'}ms`
       - default/unparseable: return raw line

    Create `src/tests/gemini.test.ts` mirroring claude.test.ts structure with equivalent test cases for parseGeminiEvent and formatGeminiEvent.

    Create `src/hooks/useGeminiTask.ts` mirroring useClaudeTask.ts:
    - Import `commands` from bindings, `formatGeminiEvent` from lib/gemini, `emitProcessOutput` + `useProcessStore` from useProcess
    - Same state: isRunning, hasApiKey, rateLimitWarning, silentFailure
    - checkApiKey calls `commands.hasGeminiApiKey()` (not Claude)
    - spawnOnce:
      - Creates Channel<OutputEvent>
      - On stdout: capture result events (look for `parsed.type === 'result'`), format via `formatGeminiEvent`
      - On stderr: detect rate limit via same string patterns (429, RESOURCE_EXHAUSTED, quota, rate limit)
      - On exit: validate via `commands.validateGeminiResult(lastResultJson)`, update process store status
      - Spawn via `commands.spawnGeminiTask(prompt, projectDir, channel)`
      - Register in process store with label `gemini: ${prompt.slice(0, 60)}`
    - spawnTask: Same retry loop with same policy (3 retries, 5s base, 60s max)
    - Return same API shape as useClaudeTask
