  <behavior>
    - parse_stream_line parses init event: `{"type":"init","session_id":"abc","model":"gemini-2.5-pro"}` -> GeminiStreamEvent::Init with session_id, model
    - parse_stream_line parses message event: `{"type":"message","role":"model","content":"Hello"}` -> GeminiStreamEvent::Message with role, content
    - parse_stream_line parses tool_use event: `{"type":"tool_use","tool_name":"Bash","tool_id":"t1","parameters":{"command":"ls"}}` -> GeminiStreamEvent::ToolUse with tool_name, tool_id, parameters
    - parse_stream_line parses tool_result event: `{"type":"tool_result","tool_id":"t1","status":"success","output":"file.txt"}` -> GeminiStreamEvent::ToolResult
    - parse_stream_line parses result event: `{"type":"result","status":"completed","response":"Done","stats":{"total_tokens":500}}` -> GeminiStreamEvent::Result with stats
    - parse_stream_line parses error event: `{"type":"error","message":"API error"}` -> GeminiStreamEvent::Error
    - parse_stream_line returns None for non-JSON lines and empty lines
    - validate_result returns Ok for result with non-empty response and non-error status
    - validate_result returns Err for empty response (silent failure)
    - validate_result returns Err for status="error"
    - validate_result returns Err for Error event
    - validate_result returns Err for non-result event
    - detect_rate_limit detects "429", "RESOURCE_EXHAUSTED", "quota", "Too Many Requests", "rate limit"
    - detect_rate_limit returns None for normal lines
    - build_command produces cmd="gemini", args=["-p", prompt, "--output-format", "stream-json", "--yolo"], env=[("GEMINI_API_KEY", key), ("NO_COLOR", "1")]
    - RetryPolicy::default_gemini() returns max_retries=3, base_delay=5000, max_delay=60000
    - delay_for_attempt doubles each attempt, capped at max_delay
  </behavior>
  <action>
    Create `src-tauri/src/adapters/gemini.rs` mirroring claude.rs structure exactly but with Gemini-specific types:

    1. **GeminiStreamEvent enum** (serde tagged on "type"):
       - Init { session_id: Option<String>, model: Option<String>, timestamp: Option<String> }
       - Message { role: Option<String>, content: Option<String>, timestamp: Option<String> } -- NOTE: Gemini content is a plain string, NOT Vec<ContentBlock> like Claude
       - ToolUse { tool_name: Option<String>, tool_id: Option<String>, parameters: Option<serde_json::Value>, timestamp: Option<String> }
       - ToolResult { tool_id: Option<String>, status: Option<String>, output: Option<String>, timestamp: Option<String> }
       - Error { message: Option<String>, timestamp: Option<String> }
       - Result { status: Option<String>, response: Option<String>, stats: Option<GeminiStats>, timestamp: Option<String> }

    2. **GeminiStats struct**: total_tokens, input_tokens, output_tokens, duration_ms (all Option<u64>), tool_calls: Option<u32>

    3. **GeminiCommand struct** + **build_command**: Same shape as ClaudeCommand. cmd="gemini", args=["-p", prompt, "--output-format", "stream-json", "--yolo"], env=[("GEMINI_API_KEY", api_key), ("NO_COLOR", "1")]. The --yolo flag is required for headless tool execution (Pitfall 4).

    4. **parse_stream_line**: Same as Claude -- trim, return None for empty, serde_json::from_str().ok()

    5. **validate_result**: Check response non-empty, status not "error", return Err for Error events and non-result events. Do NOT check num_turns or is_error (Claude-specific fields per Pitfall 6).

    6. **detect_rate_limit**: Match "429", "RESOURCE_EXHAUSTED", "quota", "Too Many Requests", "rate limit" (case-insensitive via to_lowercase). Per Pitfall 3, these differ from Claude's patterns.

    7. **RetryPolicy::default_gemini()**: Same values as Claude (3 retries, 5s base, 60s max).

    8. **Unit tests**: Mirror claude.rs test structure -- test each event type parsing, validation cases, rate limit detection, command builder, retry policy.
