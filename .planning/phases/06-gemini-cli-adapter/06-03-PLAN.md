    3. **RetryPolicy struct** (shared, moved here from each adapter):
       ```rust
       pub struct RetryPolicy {
           pub max_retries: u32,
           pub base_delay_ms: u64,
           pub max_delay_ms: u64,
       }
       impl RetryPolicy {
           pub fn delay_for_attempt(&self, attempt: u32) -> u64 {
               let delay = self.base_delay_ms.saturating_mul(2u64.saturating_pow(attempt));
               delay.min(self.max_delay_ms)
           }
       }
       ```

    4. **ToolAdapter trait**:
       ```rust
       pub trait ToolAdapter {
           /// Build the CLI command for this tool with the given prompt, working directory, and API key.
           fn build_command(&self, prompt: &str, cwd: &str, api_key: &str) -> ToolCommand;

           /// Parse a single NDJSON line from the tool's output stream.
           /// Returns a JSON string representation of the parsed event for cross-adapter validation,
           /// or None if the line is not valid JSON.
           fn parse_stream_line(&self, line: &str) -> Option<String>;

           /// Validate a result JSON string for silent failures.
           /// Returns Ok(()) if the result is valid, Err(description) if it indicates a failure.
           fn validate_result_json(&self, result_json: &str) -> Result<(), String>;

           /// Detect rate-limit or quota errors in a stderr/stdout line.
           fn detect_rate_limit(&self, line: &str) -> Option<RateLimitInfo>;

           /// Get the default retry policy for this tool.
           fn retry_policy(&self) -> RetryPolicy;

           /// Human-readable name for this tool (e.g., "Claude Code", "Gemini CLI").
           fn name(&self) -> &str;
       }
       ```

    IMPORTANT design notes:
    - The trait uses `&self` methods so adapters can be instantiated as unit structs (zero-cost).
    - `parse_stream_line` returns `Option<String>` (the raw JSON line if parseable) rather than a generic event type, because Claude and Gemini have different event enums. The specific event type parsing stays in each adapter module for type-safe access. The trait method confirms "this line is valid JSON for this adapter."
    - `validate_result_json` accepts a `&str` JSON and internally parses + validates, matching how the IPC commands already work (they receive a JSON string from the frontend).
    - Keep `pub mod claude; pub mod gemini;` at the top of the file.
    - Do NOT remove the standalone functions from claude.rs/gemini.rs yet -- they remain as the internal implementation that the trait methods delegate to. This preserves all existing tests.
