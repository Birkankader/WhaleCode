<task type="auto" tdd="true">
  <name>Task 1: Diff generation module and new types</name>
  <files>src-tauri/src/worktree/diff.rs, src-tauri/src/worktree/models.rs, src-tauri/src/worktree/mod.rs</files>
  <behavior>
    - Test: generate_worktree_diff returns FileDiff entries for files modified on a branch vs default
    - Test: FileDiff.status is "added" for new files, "modified" for changed files, "deleted" for removed files
    - Test: FileDiff.patch contains unified diff text with +/- lines
    - Test: additions/deletions counts are correct
    - Test: empty diff (no changes) returns empty files vec
    - Test: selective_merge applies only listed files from branch tree to default branch
    - Test: selective_merge with empty accepted_files list makes no changes
    - Test: selective_merge handles nested paths (e.g., src/commands/foo.rs)
  </behavior>
  <action>
1. Add new types to `src-tauri/src/worktree/models.rs`:
   - `FileDiff { path: String, status: String, old_path: Option<String>, patch: String, additions: u32, deletions: u32 }` with Debug, Clone, Serialize, Type derives
   - `WorktreeDiffReport { branch_name: String, default_branch: String, files: Vec<FileDiff>, total_additions: u32, total_deletions: u32 }` with same derives
   - Use `String` for path fields (not PathBuf) and `u32` for numeric fields per specta constraints

2. Create `src-tauri/src/worktree/diff.rs` with:
   - `pub fn generate_worktree_diff(repo_path: &Path, branch_name: &str) -> Result<WorktreeDiffReport, String>` that:
     - Opens repo, finds default branch using same logic as commands/worktree.rs `find_default_branch`
     - Resolves both branch commits and their trees
     - Uses `repo.diff_tree_to_tree(Some(&default_tree), Some(&branch_tree), Some(&mut opts))` with 3 context lines
     - Iterates deltas, extracts path/status/old_path for each
     - Uses `git2::Patch::from_diff(&diff, idx)` to get per-file patch text via `patch.to_buf()`
     - Maps `git2::Delta::Added/Deleted/Modified/Renamed` to string status
     - Counts additions/deletions by scanning patch lines starting with '+'/'-' (excluding header lines '+++'/'---')
     - Truncates patch text at 50KB per file with "[diff truncated]" message (Pitfall 4)
   - `pub fn selective_merge(repo_path: &Path, branch_name: &str, accepted_files: &[String]) -> Result<(), String>` that:
     - Opens repo, resolves default branch and both commits/trees
     - Uses `repo.index()` approach (NOT TreeBuilder) to handle nested paths (Pitfall 1):
       - Read default branch tree into a new in-memory index
       - For each accepted file: if file exists in branch tree, update/add entry in index from branch tree; if file was deleted in branch (exists in default but not branch), remove from index
       - Write the index to a new tree OID
     - Creates a merge commit on default branch with the selective tree, referencing both branch and default commits as parents
     - Updates HEAD and working directory

3. Add `pub mod diff;` to `src-tauri/src/worktree/mod.rs`

Reuse test helpers from conflict.rs tests (create_test_repo_with_file, create_branch_with_change, checkout_default_branch) -- copy them into diff.rs tests module since they are in #[cfg(test)] and not publicly exported.

  </verify>
  <done>generate_worktree_diff returns correct per-file diffs with patch text, status, and line counts. selective_merge applies only accepted files including nested paths. All tests pass.</done>
