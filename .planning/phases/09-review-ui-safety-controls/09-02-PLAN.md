   - This allows tasks to transition from 'completed' to 'review' when user opens diff view

2. Extend `src/hooks/useWorktree.ts` with two new methods:
   - `getWorktreeDiff(branchName: string)`: calls `commands.getWorktreeDiff(projectDir, branchName)`, returns `WorktreeDiffReport | null`, sets loading/error state
   - `selectiveMerge(branchName: string, acceptedFiles: string[])`: calls `commands.mergeWorktree(projectDir, branchName, acceptedFiles)`, on success calls `refreshWorktrees()` and clears conflicts
   - Update existing `mergeWorktree` to pass `null` as third arg for backward compatibility: `commands.mergeWorktree(projectDir, branchName, null)`
   - Add `diffReport` state: `useState<WorktreeDiffReport | null>(null)` and expose it in return

3. Create `src/components/review/FileDiffView.tsx`:
   - Takes `patch: string` prop
   - Splits patch into lines, renders each in a `<div>` with monospace font
   - Line coloring by prefix: `+` lines get green text + green bg/20, `-` lines get red text + red bg/20, `@@` lines get cyan, `diff`/`index`/`---`/`+++` header lines get muted zinc-500, context lines get zinc-300
   - Uses `whitespace-pre` for preserving indentation
   - If patch is empty, show "No diff available" message
   - Tailwind classes only, no external CSS

4. Create `src/components/review/DiffReview.tsx`:
   - Props: `projectDir: string, branchName: string, taskId: string, onClose: () => void`
   - On mount: call `getWorktreeDiff(branchName)` from useWorktree hook
   - State: `acceptedFiles: Set<string>` initialized with ALL file paths (default accept all)
   - Layout: full-height flex panel replacing terminal area
     - Left sidebar (w-72): scrollable file list with checkboxes
       - Each file shows: checkbox, status icon (+/M/D/R with color), file path (truncated), +N -N counts
       - Click file name to select it for viewing in the main area
       - "Select All" / "Deselect All" buttons at top
     - Main area: FileDiffView for the currently selected file
     - Bottom bar: summary ("N files accepted, M rejected"), "Merge Accepted" button (primary), "Discard All" button (danger/secondary)
   - "Merge Accepted" button:
     - Shows confirmation dialog: "Merge N files? M files will be discarded."
     - On confirm: calls `selectiveMerge(branchName, Array.from(acceptedFiles))`
     - On success: calls `onClose()`
   - "Discard All" button:
     - Shows confirmation: "Discard all changes from this task?"
     - On confirm: calls `cleanupWorktrees()` or removes the specific worktree, then `onClose()`
   - Loading state: show spinner while diff is being fetched
   - Error state: show error message with retry button
   - Use window.confirm() for confirmation dialogs (simple, sufficient for v1)
