<task type="auto" tdd="true">
  <name>Task 1: Implement ConflictDetector with merge_trees conflict simulation and auto-commit</name>
  <files>src-tauri/src/worktree/conflict.rs, src-tauri/src/worktree/mod.rs</files>
  <behavior>
    - detect_conflicts(repo_path, branch_a, branch_b) returns ConflictReport with has_conflicts=false when branches don't overlap
    - detect_conflicts returns ConflictReport with has_conflicts=true and file paths when branches modify the same file differently
    - detect_conflicts handles case where branches have no common ancestor (returns error)
    - auto_commit_worktree(worktree_path) commits all uncommitted changes with "whalecode: auto-commit" message
    - auto_commit_worktree returns Ok(false) if no uncommitted changes exist
    - auto_commit_worktree returns Ok(true) if changes were committed
    - Tests use temporary git repos with known file structures for deterministic conflict testing
  </behavior>
  <action>
    1. Create `src-tauri/src/worktree/conflict.rs` with:

       - `pub fn detect_conflicts(repo_path: &Path, branch_a: &str, branch_b: &str) -> Result<ConflictReport, String>`:
         - Open repo via `Repository::open(repo_path)`
         - Resolve both branches to commits via `repo.revparse_single(branch).peel_to_commit()`
         - Find merge base via `repo.merge_base(commit_a.id(), commit_b.id())`
         - Get ancestor tree from merge base commit
         - Get tree_a and tree_b from branch commits
         - Run `repo.merge_trees(&ancestor, &tree_a, &tree_b, None)` for read-only three-way merge
         - Check `index.has_conflicts()` and iterate `index.conflicts()` to extract file paths
         - Build and return ConflictReport

       - `pub fn auto_commit_worktree(worktree_path: &Path) -> Result<bool, String>`:
         - Open repo at worktree_path
         - Get repo status; if no modified/new/deleted files, return Ok(false)
         - Stage all changes via `repo.index().add_all(["*"], ADD_DEFAULT, None)`
         - Write index tree via `index.write_tree()`
         - Create commit with message "whalecode: auto-commit task changes"
         - Return Ok(true)

    2. Update `src-tauri/src/worktree/mod.rs` to add `pub mod conflict;`

    3. Write tests in `conflict.rs` using `#[cfg(test)]` module:
       - Helper: `create_test_repo_with_file(name, content)` — creates tempdir repo with initial commit containing a file
       - Helper: `create_branch_with_change(repo, branch_name, file, content)` — creates a branch and modifies a file
       - Test: no_conflict_when_different_files — branch_a modifies file1, branch_b modifies file2, no conflict
       - Test: conflict_when_same_file_different_changes — both branches modify same file differently, has_conflicts=true, file path in report
       - Test: auto_commit_with_changes — create uncommitted file, auto_commit returns true, file is committed
       - Test: auto_commit_no_changes — clean worktree, auto_commit returns false
  </action>
  <verify>
    <automated>cd src-tauri && cargo test worktree::conflict -- --nocapture</automated>
  </verify>
  <done>ConflictDetector correctly identifies file-level conflicts between worktree branches using merge_trees. Auto-commit captures uncommitted tool changes before conflict detection.</done>

  <name>Task 2: Create IPC commands, integrate worktree into Claude spawn flow, and wire startup cleanup</name>
  <files>src-tauri/src/commands/worktree.rs, src-tauri/src/commands/mod.rs, src-tauri/src/commands/claude.rs, src-tauri/src/lib.rs</files>
  <action>
    1. Create `src-tauri/src/commands/worktree.rs` with 5 IPC commands:

       - `create_worktree(task_id: String, project_dir: String) -> Result<WorktreeEntry, String>`:
         - Wraps `WorktreeManager::new(project_dir).create_for_task(&task_id)` in `spawn_blocking`
         - Returns the WorktreeEntry for frontend tracking

       - `check_worktree_conflicts(project_dir: String, branch_a: String, branch_b: String) -> Result<ConflictReport, String>`:
         - First calls `auto_commit_worktree` on both worktree paths (resolve from branch names)
         - Then calls `detect_conflicts(project_dir, &branch_a, &branch_b)`
         - Returns ConflictReport

       - `merge_worktree(project_dir: String, branch_name: String) -> Result<(), String>`:
         - Check conflicts against main first — if conflicts, return Err (SAFE-04: pre-merge gate)
         - Also check against all other active whalecode branches (list_worktrees, resolve each to branch)
         - If no conflicts, perform fast-forward merge of branch into main/master
         - Clean up the worktree after successful merge

       - `cleanup_worktrees(project_dir: String) -> Result<Vec<String>, String>`:
         - Wraps `WorktreeManager::cleanup_stale_worktrees()` in `spawn_blocking`

       - `list_worktrees(project_dir: String) -> Result<Vec<String>, String>`:
         - Wraps `WorktreeManager::list_worktrees()` in `spawn_blocking`

       All commands use `#[tauri::command]` and `#[specta::specta]` decorators.

    2. Update `src-tauri/src/commands/mod.rs`:
       - Add `pub mod worktree;`
       - Add pub use for all 5 worktree commands

    3. Modify `src-tauri/src/commands/claude.rs` `spawn_claude_task`:
       - After building the Claude command, create a worktree: `WorktreeManager::new(project_dir).create_for_task(&task_id)`
       - Override `cmd.cwd` with the worktree path (worktree_entry.path)
       - The tool now runs inside the isolated worktree instead of the project directory
       - Note: task_id is generated by spawn_with_env; generate the task_id (uuid) BEFORE spawning so it can be used for both worktree creation and process tracking. Pass it through or refactor spawn_with_env to accept an optional task_id.

    4. Register all 5 new IPC commands in `src-tauri/src/lib.rs`:
       - Add to the `collect_commands!` macro: `create_worktree, check_worktree_conflicts, merge_worktree, cleanup_worktrees, list_worktrees`
       - Add corresponding `use` imports from commands module

    5. Wire automatic startup cleanup in `src-tauri/src/lib.rs`:
       - Inside the existing `tauri::Builder::default().setup(|app| { ... })` closure, after the current setup logic, add worktree cleanup:
         ```rust
         // Clean up stale worktrees from previous crashed sessions
         // This satisfies ROADMAP success criterion 4: abandoned worktrees cleaned on next launch
         if let Some(project_dir) = /* resolve current project dir from app state or config */ {
             let manager = crate::worktree::manager::WorktreeManager::new(project_dir);
             match manager.cleanup_stale_worktrees() {
                 Ok(cleaned) => {
                     if !cleaned.is_empty() {
                         log::info!("Cleaned {} stale worktrees on startup: {:?}", cleaned.len(), cleaned);
                     }
                 }
                 Err(e) => {
                     log::warn!("Failed to cleanup stale worktrees on startup: {}", e);
                 }
             }
         }
         ```
       - The cleanup must NOT block app startup or cause a crash — errors are logged and swallowed
       - If the project directory is not yet known at setup time (user hasn't opened a project), defer cleanup: store a flag in AppState and run cleanup on the first `spawn_claude_task` call instead. Either approach satisfies the requirement as long as cleanup happens automatically without user action.
