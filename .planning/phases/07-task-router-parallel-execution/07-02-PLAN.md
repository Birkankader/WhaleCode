**stores/taskStore.ts:** Create a Zustand store for tracking dispatched tasks with routing metadata.

```typescript
export type ToolName = 'claude' | 'gemini';
export type TaskStatus = 'pending' | 'routing' | 'running' | 'completed' | 'failed' | 'waiting';

export interface TaskEntry {
  taskId: string;
  prompt: string;
  toolName: ToolName;
  status: TaskStatus;
  description: string;       // prompt truncated to 60 chars for display
  startedAt: number | null;  // Date.now() when dispatched
  dependsOn: string | null;  // taskId of dependency (optional, manual)
}
```

Store interface:
- `tasks: Map<string, TaskEntry>` -- all dispatched tasks
- `addTask(entry: TaskEntry): void` -- adds a task to the map
- `updateTaskStatus(taskId: string, status: TaskStatus): void`
- `removeTask(taskId: string): void`
- `getRunningTaskForTool(toolName: ToolName): TaskEntry | undefined` -- find running task for a tool

**hooks/useTaskDispatch.ts:** Create unified dispatch hook that:

1. Exposes `suggestTool(prompt: string): Promise<RoutingSuggestion | null>` -- calls `commands.suggestTool(prompt)`. Returns null on error.

2. Exposes `dispatchTask(prompt: string, projectDir: string, toolName: ToolName, dependsOn?: string): Promise<string | null>`:
   - Generate a temporary taskId (crypto.randomUUID or fallback).
   - Add task to taskStore with status 'pending'.
   - If `dependsOn` is set, check if that task is still running in useProcessStore. If running, set status to 'waiting' and await completion (subscribe to store changes). If dependency failed, set this task to 'failed' and return null.
   - Create a new `Channel<OutputEvent>`.
   - Set up channel.onmessage to route events through emitProcessOutput (same pattern as useClaudeTask/useGeminiTask). On 'stdout', format through the correct formatter (formatClaudeEvent or formatGeminiEvent based on toolName). On 'exit', update both process store and task store status.
   - Call `commands.dispatchTask(prompt, projectDir, toolName, channel)`.
   - On success: register in useProcessStore (like existing hooks do), update taskStore with real taskId and status 'running', set startedAt.
   - On error: update taskStore status to 'failed', return null.
   - Return the taskId.

3. Exposes `isToolBusy(toolName: ToolName): boolean` -- checks useProcessStore for a running process with matching tool_name in cmd prefix (e.g., cmd starts with "claude:" or "gemini:").

Import formatClaudeEvent from '../lib/claude' and formatGeminiEvent from '../lib/gemini'. Import emitProcessOutput from './useProcess'.

NOTE: Keep useClaudeTask and useGeminiTask as-is. The new useTaskDispatch composes their patterns but calls dispatch_task directly (which internally calls the same spawn functions). This avoids breaking existing hooks.
