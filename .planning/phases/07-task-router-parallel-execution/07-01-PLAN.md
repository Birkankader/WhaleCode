**state.rs:** Add three fields to `ProcessEntry`:
- `pub tool_name: String` -- "claude", "gemini", or "test" for test processes
- `pub task_description: String` -- user's prompt truncated to 120 chars
- `pub started_at: i64` -- unix timestamp millis from `chrono::Utc::now().timestamp_millis()`

Update the existing `ProcessEntry` construction sites. In `src-tauri/src/process/manager.rs`, the `spawn_with_env` function creates ProcessEntry -- add default values: `tool_name: "test".to_string()`, `task_description: "".to_string()`, `started_at: chrono::Utc::now().timestamp_millis()`. This ensures backwards compatibility for existing spawn_process calls.

**commands/router.rs:** Create new file with two IPC commands:

1. `suggest_tool(prompt: String, state: tauri::State<'_, AppState>) -> Result<RoutingSuggestion, String>`:
   - Lock state, iterate processes to find running ones. Determine claude_busy by checking `p.tool_name == "claude" && matches!(p.status, ProcessStatus::Running)`. Same for gemini_busy.
   - Call `TaskRouter::suggest(&prompt, claude_busy, gemini_busy)` and return result.

2. `dispatch_task(prompt: String, project_dir: String, tool_name: String, on_event: Channel<OutputEvent>, state: tauri::State<'_, AppState>, context_store: tauri::State<'_, ContextStore>) -> Result<String, String>`:
   - First, check if the requested tool already has a running process (max 1 per tool). Lock state, check processes for matching tool_name + Running status. If busy, return `Err(format!("{} is already running a task", tool_name))`.
   - Match on tool_name: "claude" -> call `spawn_claude_task(prompt, project_dir, on_event, state, context_store).await`, "gemini" -> call `spawn_gemini_task(prompt, project_dir, on_event, state, context_store).await`, _ -> return Err.
   - After spawn succeeds, re-lock state and update the ProcessEntry for the returned task_id: set `tool_name` and `task_description` (prompt truncated to 120 chars). The started_at is already set by spawn_with_env.

Both commands need `#[tauri::command]` and `#[specta::specta]` attributes.

**commands/mod.rs:** Add `pub mod router;` and `pub use router::{suggest_tool, dispatch_task};`.

**lib.rs:** Add `suggest_tool` and `dispatch_task` to the `collect_commands!` macro. This will regenerate TypeScript bindings on next `cargo build`.

Run `cargo build` to regenerate bindings and verify compilation.
