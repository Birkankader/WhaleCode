  <name>Task 1: Implement build_context_preamble function</name>
  <files>src-tauri/src/context/injection.rs, src-tauri/src/context/mod.rs</files>
  <behavior>
    - Test: Empty event history produces empty string
    - Test: Single event produces preamble with "Recent Project Context" header, tool name, summary, and file list
    - Test: Multiple events are listed in reverse chronological order (most recent first)
    - Test: Preamble is truncated at max_chars limit (entry that would exceed limit is not included)
    - Test: Preamble includes file paths from each event
    - Test: Events with no summary show "no summary" placeholder
    - Test: max_events parameter limits number of events included
  </behavior>
  <action>
    1. Add `pub mod injection;` to src-tauri/src/context/mod.rs

    2. Create src-tauri/src/context/injection.rs with:

    `build_context_preamble(conn: &Connection, project_dir: &str, max_events: u32, max_chars: usize) -> Result<String, rusqlite::Error>`
    - Call queries::get_recent_events(conn, project_dir, max_events)
    - If empty, return Ok(String::new())
    - Build formatted string starting with "## Recent Project Context\n\n"
    - For each event, format as:
      ```
      - [{created_at}] {tool_name} ({event_type}): {summary or "no summary"}
        Files: {comma-separated file paths}
      ```
    - Check preamble.len() + entry.len() > max_chars before adding each entry; break if exceeded
    - Return the assembled preamble

    Note: Default recommended values for callers: max_events=5, max_chars=2000. These are NOT hardcoded in the function -- they are parameters. The caller (spawn_claude_task) will use these defaults.

    Tests should create an in-memory Connection, run migrations, insert test events via queries::record_task_completion, then verify build_context_preamble output.
  </action>
  <verify>
    <automated>cd src-tauri && cargo test context::injection -- --nocapture</automated>
  </verify>
  <done>build_context_preamble produces bounded, formatted context from recent events. Empty history produces empty string. Character limit prevents unbounded growth.</done>
</task>

    1. Modify `spawn_claude_task` to accept an additional `tauri::State<'_, crate::context::store::ContextStore>` parameter named `context_store`.

    2. After retrieving the API key and before calling `build_command`, inject context:
       ```rust
       // Build context preamble from recent project history
       let context_store_clone = context_store.inner().clone();
       let project_dir_clone = project_dir.clone();
       let context_preamble = tokio::task::spawn_blocking(move || {
           context_store_clone.with_conn(|conn| {
               crate::context::injection::build_context_preamble(conn, &project_dir_clone, 5, 2000)
           })
       }).await.map_err(|e| format!("Context injection failed: {}", e))??;

       let full_prompt = if context_preamble.is_empty() {
           prompt
       } else {
           format!("{}\n\n---\nUser task:\n{}", context_preamble, prompt)
       };
       ```

    3. Pass `full_prompt` (instead of `prompt`) to `build_command`.

    4. For the ContextStore to be cloneable for spawn_blocking, add `Clone` to ContextStore by wrapping conn in `Arc<Mutex<Connection>>` instead of plain `Mutex<Connection>`. Update store.rs accordingly:
       - Change `conn: Mutex<Connection>` to `conn: Arc<Mutex<Connection>>`
       - Derive or implement `Clone` for ContextStore
       - Update `with_conn` to use `self.conn.lock()`

    Important: This change means spawn_claude_task now has 4 state parameters (prompt, project_dir, on_event, state + context_store). Tauri supports multiple managed states -- each `State<T>` is resolved independently.

    5. Verify `cargo build` succeeds and `cargo test` passes (existing tests should be unaffected since ContextStore API is unchanged).
