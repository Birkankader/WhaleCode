    - Test: record_task_completion with empty files_changed list inserts event but no file_changes rows
    - Test: get_recent_file_changes returns changes ordered by created_at DESC
    - Test: get_recent_file_changes respects limit parameter
    - Test: get_recent_file_changes filters by project_dir (changes from other projects not returned)
    - Test: get_recent_events returns events with their associated file paths
    - Test: get_recent_events respects limit parameter
    - Test: extract_file_changes_from_claude_events finds Write and Edit tool_use events
    - Test: extract_file_changes_from_claude_events returns empty vec for events with no file operations
  </behavior>
  <action>
    1. Add `pub mod queries;` to src-tauri/src/context/mod.rs

    2. Create src-tauri/src/context/queries.rs with these functions:

    `record_task_completion(conn: &Connection, task_id, tool_name, event_type, prompt, summary, project_dir, duration_ms: Option<u64>, cost_usd: Option<f64>, files_changed: &[(String, String)]) -> Result<i64, rusqlite::Error>`
    - Use `conn.unchecked_transaction()` for atomicity
    - INSERT into context_events, get last_insert_rowid
    - INSERT each file_change with the event_id
    - Commit transaction, return event_id

    `get_recent_file_changes(conn: &Connection, project_dir: &str, limit: u32) -> Result<Vec<FileChangeRecord>, rusqlite::Error>`
    - JOIN file_changes with context_events
    - WHERE project_dir = ?
    - ORDER BY file_changes.created_at DESC
    - LIMIT ?

    `get_recent_events(conn: &Connection, project_dir: &str, limit: u32) -> Result<Vec<(ContextEvent, Vec<String>)>, rusqlite::Error>`
    - Query context_events WHERE project_dir = ? ORDER BY created_at DESC LIMIT ?
    - For each event, query file_changes WHERE event_id = ? to get file paths
    - Return tuples of (event, file_paths)

    `extract_file_changes_from_claude_events(events: &[crate::adapters::claude::ClaudeStreamEvent]) -> Vec<(String, String)>`
    - Iterate events, match on ToolUse variant
    - "Write" tool -> ("created", file_path from input.file_path)
    - "Edit" tool -> ("modified", file_path from input.file_path)
    - Return collected (file_path, change_type) pairs

    Note: All tests should use `Connection::open_in_memory()` with migrations run, NOT file-based databases. This avoids temp file cleanup issues.
  </action>

  </verify>
