    - Test: db_path_for_project produces deterministic hash-based path
    - Test: db_path_for_project produces different paths for different project dirs
  </behavior>
  <action>
    1. Add to src-tauri/Cargo.toml dependencies:
       ```toml
       rusqlite = { version = "0.38", features = ["bundled"] }
       rusqlite_migration = "2.4"
       chrono = { version = "0.4", features = ["serde"] }
       ```

    2. Create src-tauri/src/context/mod.rs:
       ```rust
       pub mod migrations;
       pub mod models;
       pub mod store;
       ```

    3. Create src-tauri/src/context/models.rs with these structs (all derive Serialize, Deserialize, Debug, Clone):
       - `ContextEvent`: id (i64), task_id (String), tool_name (String), event_type (String), prompt (Option<String>), summary (Option<String>), project_dir (String), metadata (Option<String>), duration_ms (Option<u64>), cost_usd (Option<f64>), created_at (String)
       - `FileChange`: id (i64), event_id (i64), file_path (String), change_type (String), created_at (String)
       - `FileChangeRecord`: file_path (String), change_type (String), tool_name (String), summary (Option<String>), created_at (String) -- for query results joining events+changes

    4. Create src-tauri/src/context/migrations.rs:
       - Define MIGRATIONS as a slice of `rusqlite_migration::M` entries
       - Migration 1: CREATE TABLE context_events (...) and CREATE TABLE file_changes (...) with indexes, exactly as specified in RESEARCH.md schema
       - Export `run_migrations(conn: &mut Connection) -> Result<(), rusqlite_migration::Error>`

    5. Create src-tauri/src/context/store.rs:
       - `ContextStore` struct with `conn: Mutex<Connection>`
       - `ContextStore::new(db_path: &Path) -> Result<Self, rusqlite::Error>`: create parent dirs with `create_dir_all`, open connection, set WAL mode, enable foreign keys, run migrations, return Self
       - `pub fn db_path_for_project(app_data_dir: &Path, project_dir: &str) -> PathBuf`: hash project_dir with DefaultHasher, return `app_data_dir/contexts/{hash:x}.db`
       - Use `std::sync::Mutex` (not tokio) per project decision [Research]

    Note: Use `conn.pragma_update(None, "journal_mode", "WAL")` and `conn.pragma_update(None, "foreign_keys", "ON")`.
